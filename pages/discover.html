Discover Drops

Make your Bitcoin artifact mean something everywhere.

Drops is a compact, immutable artifact layer for Bitcoin L1. It gives an inscription-style record an exact body, a creator key, a canonical ID, and a proof that lives with the confirmed Bitcoin transaction.

Bitcoin permanence, made usable

One record. One meaning. No mutable profile to rewrite.

If you are new to Bitcoin artifacts, the value is simple: a Drop lets you point to a record and say, "this exact thing is confirmed on Bitcoin." Anyone can later check the same record from Bitcoin history.

01

Born on Bitcoin L1

Drops uses a canonical OP_DROP leaf and Taproot commitment. Your proof belongs to the Bitcoin transaction, not a separate network.

02

Easy to recognize

Every Drop has a portable canonical ID derived from the reveal transaction and input index. Wallets and indexers can speak the same language.

03

Ready for what comes next

Use the same proof foundation for compact inscriptions, op-drop fungible token records such as $DROP, the first token on op-drop, and Drops Pacts smart agreements.

A foundation, not a walled garden

Build artifacts, tokens, and agreements with one clear proof story.

Drops stays deliberately compact so people can understand the record they are receiving. That same clear proof can support an inscription, $DROP, the first token on op-drop, or a Pact agreement without changing what Bitcoin confirmed.

Drops inscriptions: compact identity-rich artifacts that are easy to find and verify.
op-drop tokens: strict fungible token events with separate token parsing.
Drops Pacts: Bitcoin L1 smart agreements with visible terms and verified state transitions.

BIP-110 ready

Compatible where it matters. Strict where it counts.

Drops recognizes the registered bip110-op-drop compatibility carrier through its own strict decoder. That lets products support compatible records without blurring generic artifacts, token events, and Pact data into one ambiguous format.
