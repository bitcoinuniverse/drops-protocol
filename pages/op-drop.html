op-drop and $DROP

Fungible tokens with a Bitcoin-native proof trail.

op-drop lets people create, collect, and move fungible tokens directly on Bitcoin. $DROP, the first token on op-drop, shows the model with a fixed supply and clear rules that anyone can inspect. Behind the scenes, the protocol uses canonical JSON inside a BIP-110-safe OP_DROP Taproot leaf.

A clear fungible-token layer

One strict format for deploy, mint, and transfer events on Bitcoin L1.

You do not need to understand token code to see the benefit: anyone can look at a token's supply and mint rules, and a compatible verifier only counts events Bitcoin actually confirmed. This keeps a token's story clear from launch to transfer.

Deploy

Declare the token terms

A deploy fixes a four-character lowercase ticker, maximum supply, and per-mint limit in an exact canonical payload.

Mint

Respect the visible limits

Every mint is checked against the token's declared mint limit and remaining supply before it credits a confirmed balance.

Transfer

Move value with a receipt

A transfer creates a confirmed anchor receipt. Its first confirmed spend credits the recipient transaction output.

BIP-110 ready by design

Familiar JSON. Exact rules. No ambiguous token events.

For everyday users, this means a wallet or explorer can reject lookalike or malformed token records instead of guessing what they mean. The technical rules below are what keep that experience dependable.

op-drop uses the registered bip110-op-drop carrier with a strict canonical JSON decoder. The protocol requires exact key order, compact serialization, lowercase four-character tickers, positive decimal strings, and a body that fits the 256-byte commitment bound.

The proof path

Every balance begins with a verified Bitcoin event.

When you look at an op-drop balance, you should not have to trust a number on a website. The protocol changes a balance only after it sees and verifies the matching confirmed Bitcoin event.

01Canonical event

The exact JSON expresses deploy, mint, or transfer terms.

02BIP-110 leaf

The event body and its SHA256 hash are committed through OP_DROP.

03Confirmed Bitcoin

The Taproot reveal is checked against its spent P2TR output.

04Token state

Only valid confirmed events update supply, balances, and transfer status.
